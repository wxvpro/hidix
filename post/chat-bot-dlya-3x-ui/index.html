Телеграм бот для выдачи VLESS ключей для 3x-ui с веб панелью и широким функционалом

Обзор функционала веб панели и телеграм бота

Функции веб панели, широчайший функционал:
  • Модерация пользователей
  • Реферальная программа
  • Возможность добавить несколько серверов (Нод) и тарифов к ним
  • Тарифы (Демо тарифы / Пробные тарифы / Промо тарифы)
  • Платежи (через ЮКасса и Crypto Bot, история платежей)
  • Коды оплаты, Промокоды
  • Промокоды / Коды оплаты
  • Настройка платежей 
  • Настройка оповещений в админ-телеграм боте
 Функции телеграм бота:
  • Личный кабинет (Подписки / Платежи / Баланс / Инструкции / Помощь)
  • Пробный период (выдача VLESS ключей бесплатно на демо период, если это разрешено и настроено в веб панели)
  • Тарифы (список тарифов, указаннный в веб панели)
  • Техподдержка (написать админу одной кнопкой через чат-бота)
  • Реферальная программа (возможность получать бонусы на свой баланс за привлеченных пользователей)
  • Выдача ключей через бота телеграм
  • Новости / Контакты (произвольная ссылка на ваш канал / профиль)

Пример бота: @hidix_bot 

Стоимость файлов бота и админ панели: 35.000р (установка не входит в стоимость), торг аргументированный возможен

По вопросам приобретения пишите в телеграм @ALEX_DDE

Настройка веб-панели и телеграм бота

Подготовка:

  • Арендуем сервер VPS на базе UBUNTU, делаем на отдельном серваке
  • Получаем токен у BotFather и свой ID - будет id админа бота

Подключаемся к серверу по ssh, выполняем команды

apt-get update -y

apt-get upgrade -y

sudo apt install -y software-properties-common

sudo add-apt-repository ppa:deadsnakes/ppa

sudo apt update

sudo apt install -y python3.12

apt install python3.12-venv

Открываем DB Browser (SQLite) и открываем базу данных которая находится на сервере в bot/instance, во вкладке Browse Data в Table выбираем bot_settings, вносим телеграм токен бота и admin_id и сохраняем (apply, write changes)

заливаем папку bot на сервер в папку root (PS: с мака на винду не соблюдает пути)

cd bot

python3 -m venv venv

source venv/bin/activate

pip install -r requirements.txt
  • Запускаем бота python3 main.py
  • Останавливаем бота (Ctrl+C)

Запуск настройки рефералов python support.py для редактирования в базе данных, при первом входе сделать:

Пользователи -> Общая статистика ->  Заполнить реф. коды

Выходим, после python update_db.py

Создаем автозапускаемый сервис бота:

sudo nano /etc/systemd/system/3xui-bot.service

Вставить в него скорректировав пути к файлам:

[Unit]
Description=3xui Telegram Bot
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/bot
Environment=PATH=/root/bot/venv/bin
ExecStart=/root/bot/venv/bin/python
/root/bot/main.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
sudo systemctl enable 3xui-bot.service
sudo systemctl start 3xui-bot.service
sudo systemctl status 3xui-bot.service
sudo systemctl stop 3xui-bot.service
# - останавливаем временно

Создаем API

python api_key_manager.py

выбрать пункт 1, после перезапустить и выбрать пункт 2, имя ключа и ключ сохранить

Запуск бота: (Ps: делается в venv, не забыть) sudo systemctl start 3xui-bot

Проверить работу API модуля: python -m api.start_api

Если запустился и выдал "http://0.0.0.0:8000" - останавливаем CTRL+C останавливаем бота sudo systemctl stop 3xui-bot.service

Обновляем БД: 

python update_db.py

Создаем сервис для API модуля:

sudo nano /etc/systemd/system/tgbot-api.service

Вставить в него скорректировав пути к файлам:

[Unit]
Description=tgbot-api
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/bot
Environment=PATH=/root/bot/venv/bin
ExecStart=/root/bot/venv/bin/python -m api.start_api
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
sudo systemctl enable tgbot-api.service
sudo systemctl start tgbot-api.service
sudo systemctl status tgbot-api.service
sudo systemctl stop tgbot-api.service

Если не запустилось ( такое случается после проверки ) - перезагружаем сервер. (venv) root@localhost:~/bot# reboot

После перезагрузки (или сразу) проверяем работу сервисов:

sudo systemctl status 3xui-bot.service
sudo systemctl status tgbot-api.service

Ссылка для доков по API: http://0.0.0.0:8000/docs

Если запущены, то останавливаем их:

sudo systemctl stop 3xui-bot.service
sudo systemctl stop tgbot-api.service

Перейти в папку с модулем (наименование папки)

cd /bot/module
python3 -m venv venv
source venv/bin/activate
pip install -r /root/bot/module/requirements.txt

Проверяем работу модуля:

python app.py

Выведутся ссылки, сохранить, там будет ссылка на веб панель

Создать файл сервиса:

sudo nano /etc/systemd/system/3xui-module.service
[Unit]
Description=3xui Telegram Bot
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/bot/module
Environment=PATH=/root/bot/module/venv/bin
ExecStart=/root/bot/module/venv/bin/python /root/bot/module/app.py
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
sudo systemctl enable 3xui-module.service
sudo systemctl start 3xui-module.service
sudo systemctl enable 3xui-bot.service
sudo systemctl start 3xui-bot.service
sudo systemctl enable tgbot-api.service
sudo systemctl start tgbot-api.service

Чат и веб-панель должны заработать

Ссылка для подключения API в модуле: http://ip сервера:8000/api/v1

Идем в веб панель ЧатБота
Настройки, API - заполняем данные полученные на предыдущих шагах

Настройка HTTPS для панели

  • Ставим панель 3x ui
  • Выполняем команду: x-ui
  • Генерим сертификаты: 19 > 1, запоминаем пути к сертификатам
  • далее x-ui >удаляем x-ui

Модифицируем файлы
В файле на сервере root/bot/module/app.py последние строчки делаем так (прописываем пути к сертификатам из полученных данных выше) 

if __name__ == '__main__':
    create_tables()
    app.run(
    host='0.0.0.0',
    port=443,
    ssl_context=(
        '/root/cert/домен_панели/fullchain.pem',
        '/root/cert/домен_панели/privkey.pem'
    )
)

В файле на сервере root/bot/api/start_api.py добавить путь к сертификатам

if __name__ == "__main__":
    os.makedirs('logs', exist_ok=True)
    logger.add("logs/api.log", rotation="1 day", compression="zip", 
               encoding="utf-8", format="{time} | {level} | {message}",
               level="INFO")
    
    try:
        logger.info("Запуск API сервера...")
        port = int(os.environ.get("API_PORT", 8000))
        host = os.environ.get("API_HOST", "0.0.0.0")
        
        # Без лишней запятой
        certfile = '/root/cert/домен_панели/fullchain.pem'
        keyfile = '/root/cert/домен_панели.duckdns.org/privkey.pem'
        
        logger.info(f"Запуск API на https://{host}:{port}")
        uvicorn.run(app, host=host, port=port, ssl_certfile=certfile, ssl_keyfile=keyfile)

    except Exception as e:
        logger.error(f"Ошибка при запуске API сервера: {e}")

По хорошему сменить порт и отключить пинг на домене для бота

Проверка. Все должно работать
